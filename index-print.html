01 / NAV

Navigation — original, not a top bar.

4 patterns

Marco's brief: nothing template. The nav itself should hint at the "system" metaphor — feel like an instrument, not a menu.

N1 · Compass
arms = sections

Compass dial

Floating instrument bottom-right. Each arm = section. Spins as you scroll.

Win: instrument feel, on-brand metaphor.
Risk: obscure on first visit — needs a hover label.
N2 · System rail

Living system rail

Left rail IS the System Map mini. Walking it = walking the method.

Win: nav and method are the same artifact. Reinforces brand promise on every screen.
Pairs with: Hero v1 / v3 best.
N3 · Index cards

Index cards fan

Top-right "03/10" counter; click → cards fan out as a deck of sections.

Win: playful, premium, fits the editorial side.
Risk: animation cost; not great on touch.
N4 · Marginalia

Marginalia only

Zero global nav. Side margins hold prev/next breadcrumbs like a manuscript.

Win: editorial, premium, trusts the reader.
Risk: no quick jump; needs strong hero hook.
02 / HERO

Hero — make the visitor stop.

6 concepts

"Speechless" was the brief. Each option below is dynamic and interactive — not a static render. Annotated with motion + interaction notes.

H1 · Modular Orb Engine
spline scene

Modular Orb Engine

The brief's reference. Spline 3D core + 5 rings (Market/Offer/Channel/Auto/Metrics) with floating KPI cards.

Motion: idle slow rotate · scroll-aligned rings · cursor parallax.
Interactive: hover ring → that layer lights, others dim.
Cost: high (3D + fallback).
H2 · Liquid Constellation

Liquid Constellation

3D field of named nodes — ICP, CRM, KPI, n8n, Apollo. Cursor warps the field. Threads of light show flow.

Motion: physics-soft drift · cursor gravity · scroll → re-layouts into a clean pipeline (system emerges).
Interactive: click any node = pin. Builds the diagnostic.
Why pick: the metaphor literally happens — chaos resolves into system as you read.
H3 · Vertical Cathedral

Vertical cathedral

Whole hero is full-bleed. Scrolling = descending through the system stack. Each layer renders as a 3D level.

Motion: pinned scroll · camera drops floor by floor.
Risk: heavy; mobile fallback = static stack.
H4 · Type-as-system

Type-as-system

Headline doubles as a diagram. Each word has a satellite definition wired to it. Editorial-first, low 3D cost.

Motion: on cursor proximity, satellite labels reveal · words slowly breathe.
Win: readable in 2s, premium-magazine, cheap to ship.
H5 · Cabinet of Mechanisms

Cabinet of Mechanisms

9-cell display case. Each cell is a tiny live diagram. Together they read as a working system.

Win: dense proof of capability without listing skills.
Risk: can feel busy — needs strong whitespace.
H6 · Diagnostic Scan

Diagnostic scan

Horizontal scrubber slices through a stack. Each layer = one part of Marco's system.

Motion: scrubber drives reveal · numbers on each layer animate.
Win: crystallizes the diagnostic identity.
Best paired: with Nav N2 (system rail).
shortlist · subjective

My ranking → H2 · H1 · H6 · H4 · H5 · H3

H2 (Constellation) wins on metaphor — chaos resolves into system as you scroll. H1 is the safe Spline build. H6 is the most "Marco-thinks-in-systems" of the bunch. Best to ship one + a Tweaks toggle to preview a second on the live site.

04 / METHOD

System Map — the 6-module method.

3 layouts

Diagnose → Position → Build → Launch → Measure → Iterate. The non-negotiable spine of the site.

M1 · Pipeline

Linear pipeline

Hover any node → expands a detail card below. Most legible; closest to "tell me how you work."

M2 · Orbit

Orbital system

Six modules orbit the System core. Loops back — captures iteration honestly. Most on-brand visually.

Why pick: emphasizes loop, not funnel.
M3 · Stack

Stacked layers

Reads like an OSI stack. Good if Marco wants the "system" framing literal.

05 / CAPABILITIES

Capabilities as system modules.

3 layouts

Six categories: Strategy & GTM · Sales & Outbound · Automation & Data · RevOps · Product & Execution · AI Workflows.

C1 · Module grid

3×2 module grid

Each cell is a "component of the system." Hover wakes a tiny diagram — keeps it from feeling listy.

C2 · Honeycomb

Honeycomb

Modules interlock. Visually richer; risk of feeling biotech if rendered wrong.

C3 · Editorial strip

Editorial strip

Long-form list with rules. Slowest, most readable; pairs well with H4 (type-as-system).

06 / MODES OF WORK

2 layouts

Four-column ledger

S2 · Decision tree

Decision tree

Pipeline river

08 / PROOF

Proof — metric cards with context.

2 layouts

Big numbers alone read fake. Each metric carries a sentence that reframes it as a system, not a bragging point.

PR1 · Metric ledger

Metric ledger (3×2)

Big numerator, small contextual sentence. Numbers earn their size by carrying a verb.

Rule: no number without a sentence under it.
PR2 · Timeline ribbon

Timeline ribbon

Same metrics anchored to a real career arc. Better at conveying trajectory; weaker for skim-readers.
